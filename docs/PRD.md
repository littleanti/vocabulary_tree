# 📋 PRD — 어휘력 세계수 v1

> Product Requirements Document
> Last updated: 2026-04-25
> Stage: 5단계 (1자 → N어휘 파생 / 학술 어휘 분해)

## 1. 제품 개요

**제품명**: 어휘력 세계수 — 한자 한 글자가 키워내는 어휘 나무

**한 줄 요약**: 한자(형태소) 하나를 뿌리로 둔 나무가 어휘 잎을 맺으며 자라는 모바일·태블릿 우선 학습 게임. 1일 1자 4어휘 분량으로 인지 과부하를 방지하면서, 학술 어휘를 형태소 단위로 분해·유추하는 능력을 길러낸다.

**배포 환경**: 정적 웹앱 + PWA (Service Worker 오프라인 캐시), 포트 3005

**선후행 게임**:
- 선행: `4_morpheme_detective` — 형태소 인식 보유 전제
- 후행: `6_literacy_decoder` — 문맥 속 미지 어휘 추론으로 확장

## 2. 타겟 사용자

| 사용자 그룹 | 특징 | 주요 니즈 |
|---|---|---|
| 핵심 학습자 (초3~초4) | 한자 형태소 인식은 갖춤, 학술 어휘 진입기 | 한자어 분해·유추 능력, 교과서 어휘 대비 |
| 확장 학습자 (초5~초6) | 학술 어휘 노출 증가 | 추상 어휘 분해, 누적 어휘량 확장 |
| 학부모/교사 | 학습 관리자 | 자녀 진척도 가시화, 과학습 방지 |
| 한국어 학습자 (외국인 중·고급) | 한자 기반 어휘 학습 | 형태소 단위 어휘 확장 |

## 3. 사용자 시나리오

### S1. 초3 아이의 일일 학습
- 아침에 알림 → "오늘의 한자: 水(물 수)" 푸시
- 묘목 화면 진입 → 떠다니는 음절 블록 중 `영`, `생`, `약`, `돗`을 가지에 붙임
- 가지마다 어휘 잎 생성: `수영`, `생수`, `약수`, `수돗물`
- 잎 클릭 → 의미 + 예문 + TTS 발화
- 4어휘 완료 → 시스템이 추가 학습 차단 ("내일 또 만나요!")

### S2. 초5 아이의 학술 어휘 분해 보스
- 보스 스테이지 진입 → `이등변삼각형` 어휘 등장
- `二(둘)`, `等(같을)`, `邊(변)`, `三(셋)`, `角(뿔)`, `形(모양)` 한자 카드를 순서대로 매칭
- 매칭 성공 시 어휘 의미가 한자별 색상으로 분해되어 표시
- 보상: 세계수 가지에 "학술 어휘" 골든 잎 영구 보존

### S3. 학부모의 진척 확인
- 학부모 대시보드 진입 (PIN/생년월일 보호)
- 누적 한자 수, 어휘 수, 정답률, 마지막 학습일 시각화
- SRL 큐에 쌓인 복습 대기 어휘 수 확인
- 다음 알림 시각 조정 가능

### S4. 외국인 학습자의 자기주도 학습
- 1일 1자 락(lock) 해제 옵션 (성인 모드)
- 한자 단위 검색으로 임의 한자 선택 → 파생 어휘 학습
- 의미 표시 시 영어/일본어 보조 라벨 (옵션)

## 4. 기능 요구사항

### P0 (필수 — MVP)
| ID | 기능 | 설명 |
|---|---|---|
| F1 | 묘목 시작 화면 | 오늘의 한자(루트) + "학습 시작" 버튼 |
| F2 | 음절 블록 드래그 | 떠다니는 음절을 가지 후보에 붙이기 (자성 스냅) |
| F3 | 어휘 잎 생성 | 정답 조합 시 가지 자라기 + 잎 페이드인 애니메이션 |
| F4 | 어휘 의미 카드 | 잎 클릭 → 의미·예문·TTS 발화 모달 |
| F5 | 1일 1자 4어휘 락 | 4어휘 달성 시 추가 학습 비활성화 + 다음날 안내 |
| F6 | 나무 성장 단계 | 묘목 → 청년목 → 거목 → 세계수 (누적 어휘 임계치) |
| F7 | 한자-어휘 DB | 한자 ID → 파생 어휘 + 의미 + 예문 + 학년 태그 |
| F8 | IndexedDB 영속화 | 학습 기록(한자·어휘·SRL 큐) 누적 저장 |

### P1 (확장)
| ID | 기능 | 설명 |
|---|---|---|
| F9 | SRL 스케줄러 | 에빙하우스 곡선(1·3·7·14·30일) 기반 노출 |
| F10 | 오답 미니퀴즈 | 오답 시 즉시 등장하는 객관식 4지선다 |
| F11 | 학술 어휘 분해 보스 | 다(多)형태소 어휘를 한자 단위로 분해 매칭 |
| F12 | Viewport 카메라 | 핀치줌·팬으로 나무 전체 탐색 |
| F13 | 미니맵 | 우상단 thumbnail → 탭 시 해당 가지로 점프 |
| F14 | 학부모 대시보드 | 누적 한자/어휘/정답률/SRL 잔여 시각화 |
| F15 | 푸시 알림 | 일일 학습 시각 + SRL 복습 리마인드 |
| F16 | TTS 발음 | Web Speech API로 한자·어휘 발음 |
| F17 | PWA 설치 | 홈 화면 추가 + Service Worker 오프라인 캐시 |
| F18 | 백그라운드 자동 저장 | `visibilitychange` 시 진척도 영속화 |

### P2 (향후)
- 친구/형제 어휘 나무 비교 (로컬 가족 모드)
- 한자 손글씨 입력 (Stroke 인식)
- 어휘 도감 / 컬렉션 화면 (시각적 보상)
- 교사 모드: 반 단위 진척 CSV 내보내기
- 성인 모드 락 해제 (1일 1자 제한 OFF)
- 한자 어원 애니메이션 (갑골문 → 현대 자형)
- 다국어 보조 라벨 (영/일/중)

## 5. 비기능 요구사항

| 항목 | 요구사항 |
|---|---|
| 성능 | 묘목→잎 생성 애니메이션 60fps, 가지 추가 지연 < 300ms |
| 호환성 | iOS Safari 15+, Android Chrome 100+, 데스크톱 Chrome/Edge/Safari 최신 |
| 반응형 | 320px ~ 1024px 세로 우선, 가로/태블릿 자연스럽게 폴백 |

| 오프라인 | PWA Service Worker로 첫 로드 후 네트워크 없이 작동 |
| 데이터 | IndexedDB(Dexie) — localStorage 5MB 한도 위험 회피 |
| 무장애 | TTS 미지원 시 자동 비활성화, 알림 권한 거부 시 우회 흐름 |
| 폰트 | 한자 폰트 서브셋 임베드, `font-display: swap` |
| 보안 | 학부모 대시보드 PIN/생년월일 보호, 외부 전송 없음 |
| 개인정보 | 모든 학습 데이터 디바이스 로컬, 식별자 비수집 |

## 6. 성공 지표

- **학습 유지율**: 7일 연속 접속 비율 40%+ (1일 1자 락이 일일 복귀 동인)
- **누적 어휘 도달**: 초3-4 학습자가 6개월 후 800어휘 누적 70%+
- **SRL 효과**: 14일 리마인드 어휘의 정답률 75%+ (장기 기억 정착)
- **보스 스테이지 도전**: 해금 학습자 중 학술 어휘 분해 시도율 60%+
- **학부모 대시보드 사용**: 주 1회 이상 진입 학부모 비율 30%+
- **세션 길이**: 모바일 1세션 평균 10~15분 (커리큘럼과 일치)

## 7. 범위 제한 (Out of Scope)

- **회원가입/서버 동기화 없음** — 모든 데이터는 디바이스 로컬
- **실시간 멀티플레이** — 향후 P2 후보
- **한자 손글씨 입력** — v1 범위 외 (P2)
- **백엔드/외부 API** — TTS·폰트 외 외부 의존성 없음
- **유료 결제 / 광고** — 비영리 학습 도구 포지셔닝
- **하루 학습량 락 우회** — 의도적 과학습 방지 (성인 모드는 P2에서만)

## 8. 오픈 이슈

- [ ] 한자-어휘 DB 초기 분량: 8급(50자) MVP vs. 8~6급(200자) 1차 릴리즈
- [ ] 음절 블록 정답 판정 방식: 음절 단위 매칭 vs. 한자 단위 매칭
- [ ] 나무 시각화: SVG 정적 + 애니메이션 vs. Canvas 동적 렌더
- [ ] SRL 큐 충돌 시 우선순위: 신규 학습 vs. 복습
- [ ] 알림 채널: 웹 푸시(VAPID) vs. PWA 로컬 알림 only
- [ ] 학부모 대시보드 보호 방식: PIN vs. 생년월일 vs. 둘 다
- [ ] 학년군 자동 판별: 가입 시 1회 입력 vs. 진척에 따른 동적 조정
- [ ] 어휘 의미 재구성 원칙 검증 — 교사·언어학자 자문 필요

## 9. 홈·설정·완료 화면 디자인 공통 규격

시리즈 전체의 통일감을 위해 **홈 화면(시작 화면)**, **설정 화면**, **게임 완료 화면(lock-screen / 일일 완료 화면)** 은 `1_chosung_quiz` 의 디자인 시스템을 기준으로 제작한다.

| 디자인 요소 | 규격 |
|---|---|
| 제목 폰트 | **Jua** (Google Fonts) |
| 설명·본문 폰트 | **Gowun Dodum** (Google Fonts) |
| 버튼 디자인 | 1단계 `components.css` 버튼 스타일 — 큰 라운드 버튼, CSS 변수 색상 |
| 색감 | `1_chosung_quiz/src/css/tokens.css` CSS 변수와 동일한 팔레트 사용 |
| 배경 | `--color-bg` 등 공통 토큰과 동일한 배경 색상 |

> 나무 성장 화면·학부모 대시보드 등 게임 고유 콘텐츠 화면은 각 게임 특성에 맞게 확장 가능하다.  
> 단, 시작·설정·완료(일일 완료) 화면은 공통 디자인 언어를 유지한다.
